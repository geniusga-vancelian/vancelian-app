## 1. Executive summary

Le chantier **Privy / Web3 boundary eager cleanup** est **terminé**.

| Cible | État |
|-------|------|
| Pages **read-only** principales | Pas de `PortalWeb3Boundary` eager, pas de montage Privy / Wagmi / Viem au first paint |
| Pages **transactionnelles** | Web3 uniquement sur routes tx ou **controllers d’exécution** + `PortalWeb3BoundaryLazy` |
| Layouts segment Web3 **eager** restants | **1** — `wallet/(tx)/layout.tsx` (justifié) |

Tous les layouts segment eager historiques inutiles ont été retirés : `wallet/layout`, `borrow/layout`, `wallets/layout`, `invest/vault/layout`, `invest/bundle/layout`. L’exécution on-chain passe par des surfaces lazy (swap, vault, bundle invest, Lombard open_loan, connect wallet profil, actions bundle wallet).

**Health prod (référence) :** `https://app.vancelian.finance/api/portal/health` → **200** après déploiements F6/F7.

## 7. Ne pas régresser (checklist)

- [ ] Aucun `PortalWeb3Boundary` **eager** dans les layouts read-only (`wallet/(read)`, `borrow`, `wallets`, `invest` hub, shell global).
- [ ] Setup swap / vault / bundle invest / Lombard form : pas de `usePortalTxSigner`, pas de `usePrivyLiveSession`, pas de `usePrivy` SDK au mount (guards dédiés).
- [ ] Connect wallet externe profil : **`PortalProfileExternalWalletConnect`** — lazy au clic uniquement.
- [ ] **`wallet/(tx)/layout`** : conserver — seul offender autorisé.
- [ ] Nouveau layout eager → mettre à jour `PORTAL_WEB3_BOUNDARY_KNOWN_OFFENDER_LAYOUTS` + review explicite (CI échoue si hors liste).

## 9. Architecture cible (rappel)

```text
Shell portail (read-only)
  └─ pas de PortalWeb3Providers global

Read-only routes
  └─ BFF / cache client uniquement

Transaction setup (amount, form, quote)
  └─ API/BFF — guards interdisent hooks exécution

Review / Processing / Connect action
  └─ Portal*ExecutionController ou PortalWeb3BoundaryLazy

wallet/(tx)/*
  └─ PortalWeb3Boundary eager (swap, create, deposit, withdraw)
```
