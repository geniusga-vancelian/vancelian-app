## 1. Doctrine

| Règle | Description |
|-------|-------------|
| **Privy = exécution** | SDK Web3 (Privy, Wagmi, Viem, signers) uniquement quand une transaction on-chain est en cours ou imminente (signature, polling, retry). |
| **Privy ≠ navigation** | Markets, dashboard, détails wallet/crypto, credit line, academy, invest hub ne montent pas le SDK au chargement de page. |
| **Setup read-only** | Montant, paires, preview API/BFF : pas de `usePrivy`, pas de `usePortalTxSigner`, pas de boundary eager. |
| **Review / Processing** | Boundary Web3 au plus tard à l’entrée Review (ou Processing si pas de Review, ex. Lombard). |
| **Result** | Idéalement sans SDK ; retry transactionnel = remontée boundary lazy. |

Chaîne actuelle d’une boundary **eager** :

```text
PortalWeb3Boundary
  └─ PortalWeb3Providers → ExternalWalletProvider (Wagmi + RainbowKit)
       └─ PortalAuthPrivyGate → PrivyPortalProvider
```

Pattern **lazy** déjà validé : `PortalWeb3BoundaryLazy` (dynamic import, `ssr: false`) autour des modales / actions ponctuelles.
