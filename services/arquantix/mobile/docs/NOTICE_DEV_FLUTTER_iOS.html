Notice développement — App Flutter sur iPhone

Projet source sous OneDrive, build iOS hors OneDrive (codesign). Copie de travail : ~/dev/vancelian-mobile. Adapte IP_MAC et l’ID d’appareil si besoin.

1. Après modification du code — relancer / appliquer les changements

Synchroniser OneDrive → copie locale (à chaque fois que tu modifies des fichiers dans le dépôt OneDrive) :

rsync -a --delete \
  "/Users/gael/Library/CloudStorage/OneDrive-Vancelian/Documents/vancelian-app/services/arquantix/mobile/" \
  ~/dev/vancelian-mobile/

Puis, selon ce que tu as changé :

  • Uniquement du Dart / assets (la plupart des cas) : avec flutter run déjà actif, appuie sur r pour hot reload, ou R pour hot restart.
  • pubspec.yaml ou dépendances :
cd ~/dev/vancelian-mobile
flutter pub get
cd ios && pod install && cd ..
  • Fichiers natifs iOS (ios/, plugins) : même bloc pod install ci-dessus, puis relance complète (quitte q puis relance flutter run).
  • Changement lourd / doute : relance propre :
cd ~/dev/vancelian-mobile
flutter clean
flutter pub get
cd ios && pod install && cd ..
xattr -cr .

Lancer l’app sur l’iPhone (remplace l’IP et l’ID du téléphone) :

cd ~/dev/vancelian-mobile

flutter run -d TON_ID_IPHONE \
  --dart-define=FLAVOR=dev \
  --dart-define=API_BASE_URL=http://IP_MAC:3000 \
  --dart-define=AUTH_API_BASE_URL=http://IP_MAC:8000

Exemple :

flutter run -d 00008130-001165803A88001C \
  --dart-define=FLAVOR=dev \
  --dart-define=API_BASE_URL=http://192.168.1.42:3000 \
  --dart-define=AUTH_API_BASE_URL=http://192.168.1.42:8000
Rappel : les serveurs Next (port 3000) et API FastAPI (port 8000) doivent tourner sur le Mac pour le login / le BFF. Voir la notice « Après redémarrage du Mac ».

2. Après un redémarrage du Mac — remettre l’environnement en route

  1. Docker Desktop : ouvre-le et attends qu’il soit prêt (icône dans la barre de menu).
  2. Base de données + API + Web + tâches depuis le repo Arquantix :
cd /Users/gael/Library/CloudStorage/OneDrive-Vancelian/Documents/vancelian-app/services/arquantix
make boot

Alternative si Docker / la DB est déjà OK :

cd /Users/gael/Library/CloudStorage/OneDrive-Vancelian/Documents/vancelian-app/services/arquantix
make start-api-web

Vérifier rapidement :

curl -s -o /dev/null -w "%{http_code}\n" http://127.0.0.1:8000/health
curl -s -o /dev/null -w "%{http_code}\n" http://127.0.0.1:3000

Les codes 200 (ou réponse valide pour Next) indiquent que les services écoutent.

IP du Mac pour l’iPhone (même Wi‑Fi) :

ipconfig getifaddr en0

ID de l’iPhone :

flutter devices

Synchroniser le mobile puis lancer l’app (même flux que la section 1) :

rsync -a --delete \
  "/Users/gael/Library/CloudStorage/OneDrive-Vancelian/Documents/vancelian-app/services/arquantix/mobile/" \
  ~/dev/vancelian-mobile/

xattr -cr ~/dev/vancelian-mobile

cd ~/dev/vancelian-mobile
flutter pub get
cd ios && pod install && cd ..

flutter run -d TON_ID_IPHONE \
  --dart-define=FLAVOR=dev \
  --dart-define=API_BASE_URL=http://IP_MAC:3000 \
  --dart-define=AUTH_API_BASE_URL=http://IP_MAC:8000
Astuce : si l’IP du Mac change après le redémarrage (DHCP), mets à jour uniquement les deux --dart-define=...URL... avec la nouvelle IP.

Raccourcis utiles pendant flutter run

  • r — Hot reload (rapide)
  • R — Hot restart
  • q — Quitter (arrête l’app sur l’appareil)
