Vancelian Dev Toolkit — utilisation simple

Cette boîte à outils te donne un menu interactif pour éviter de te battre avec le terminal. Elle gère le sync mobile local, le clean Flutter, CocoaPods, le lancement des serveurs, Xcode et la cible Flutter (iPhone USB, iPhone wireless, simulateur iOS).

Emplacement dans le repo : services/arquantix/tooling/vancelian_dev_toolkit.sh — les chemins mobile/, api/, web/ sont déduits automatiquement (parent du dossier tooling/).

1. Installation

Depuis la racine du dépôt (ou le dossier tooling) :

cd services/arquantix/tooling
chmod +x vancelian_dev_toolkit.sh
./vancelian_dev_toolkit.sh

Copie pratique dans le home (comme avant) :

cp services/arquantix/tooling/vancelian_dev_toolkit.sh ~/vancelian_dev_toolkit.sh
chmod +x ~/vancelian_dev_toolkit.sh
~/vancelian_dev_toolkit.sh

2. Mode recommandé au quotidien

  • Option 10 = fast launch Flutter en --profile (sélection device smart)
  • Option 11 = clean complet + launch en --profile
  • Option 12 = launch en --release
  • Option 15 = afficher flutter devices + résumé JSON

3 bis. iPhone USB, wireless et simulateur

  • Détection : flutter devices --machine + sortie texte de flutter devices pour repérer les lignes wireless ; optionnellement xcrun xcdevice list pour USB/Wi‑Fi.
  • DEVICE_ID exporté et présent dans la liste → lancement direct sur cet UDID.
  • Pas de DEVICE_ID (ou unset DEVICE_ID) → sélection intelligente : un seul iPhone réel = auto ; plusieurs = menu ; sinon simulateur ou aide Xcode.
  • DEVICE_ID_DEFAULT dans le script = tie-break si plusieurs appareils (pas un forçage si débranché).
  • PREFER_WIRELESS=1 / PREFER_USB=1 pour départager plusieurs iPhones.
  • Rapport détaillé : FLUTTER_DEVICE_SELECTION.md dans le même dossier.

4. Aliases zsh recommandés

alias vmenu="$HOME/vancelian_dev_toolkit.sh"
alias vrun="$HOME/vancelian_dev_toolkit.sh fast"
alias vclean="$HOME/vancelian_dev_toolkit.sh clean"
alias vrelease="$HOME/vancelian_dev_toolkit.sh release"

Si tu lances depuis le clone sans copie dans ~, pointe les alias vers le chemin absolu du script dans ton dépôt.

Ajoute-les dans ~/.zshrc puis fais :

source ~/.zshrc

5. Ce que fait le script

  • détecte automatiquement l’IP locale du Mac
  • utilise --profile par défaut pour un rendu iPhone réel fluide
  • ajoute TRACE_JANK=true en profile pour tes traces
  • ouvre le bon workspace Xcode : Runner.xcworkspace
  • choisit la cible Flutter (USB / wireless / simulateur) de façon smart ; DEVICE_ID prioritaire si exporté et présent
  • si aucun iPhone prêt : peut ouvrir Xcode et afficher une checklist (câble, confiance, mode développeur)
  • lance l’API avec python -m uvicorn (venv api/.venv ou api/venv prioritaire sur python3 système)
